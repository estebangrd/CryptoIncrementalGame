# Blockchain Tycoon — Store Listing Pack

Ficha de tienda lista para copiar a **App Store Connect** (iOS) y **Google Play Console** (Android) en los 7 idiomas soportados: English, Español, Português, Deutsch, Français, Italiano, Русский.

> **Para release Android-only de v1.0**: pegá únicamente **Title**, **Short description** y **Full description**. Los campos **Subtitle**, **Promotional text** y **Keywords** son exclusivos de App Store Connect (iOS) — ignoralos hasta que prepares el release iOS.

## Field reference cheatsheet

| Field | iOS (App Store Connect) | Android (Play Console) | Limit |
|---|---|---|---|
| **Title** | App Name | App name | 30 chars iOS · 30 chars Android |
| **Subtitle** | Subtitle | — | 30 chars iOS only |
| **Short description** | — | Short description | 80 chars Android only |
| **Full description** | Description | Full description | 4000 chars |
| **Keywords** | Keywords | (use full description naturally) | 100 chars iOS only, comma-separated |
| **Promotional text** | Promotional Text | — | 170 chars iOS only |

**ASO tips applied below:**
- Keywords: focus on long-tail "idle game", "incremental", "tycoon", "mining" + crypto theme. Don't repeat words from the title (iOS doesn't double-rank them).
- Subtitle: use it as a second keyword slot — different terms than the title.
- Short description (Android): first 80 chars are the highest-impact ASO surface — punchy hook + keyword.
- Full description: front-load the value proposition. First 250 chars are visible above the "more" fold.

## Submission checklist

Antes de subir cada idioma:

- [ ] **Conteo de caracteres**: la mayoría de los títulos están en 28–30 chars. Si Apple/Google rechaza, usá el fallback más corto.
- [ ] **Keywords iOS**: solo 100 chars total, sin espacios después de comas. NO repetir palabras del título.
- [ ] **Screenshots**: 5 capturas verticales por idioma (mínimo). Mostrar idioma localizado en pantalla cuando sea posible.
- [ ] **Subtitle iOS**: aparece bajo el título en la búsqueda. Usá keywords distintos a los del título.
- [ ] **Promotional text iOS**: actualizable sin re-review. Usalo para anuncios de eventos / sales / nuevas features.
- [ ] **Short description Android**: solo 80 chars. Es lo más visible en Play Store search.
- [ ] **Privacy policy URL**: requerida en ambas tiendas. Misma URL para todos los idiomas si la tenés en inglés.
- [ ] **App Store category**: iOS = Games > Simulation (secondary: Strategy). Android = Games > Simulation.
- [ ] **Content rating**: rellenar el cuestionario IARC en cada tienda. Sin violencia, lenguaje suave en narrativa endgame → debería caer en 7+/Teen.
- [ ] **In-app purchases**: localizar nombres y descripciones de IAPs en App Store Connect / Play Console (separado de esta ficha — usar la lista de boosters/packs de `src/data/translations.ts`).

## Notas para release sucesivos

- **Promotional text** se puede actualizar sin nueva review en iOS — usalo para destacar eventos temporales (flash sale, packs nuevos, finales).
- **What's new / Release notes** por versión: traducí en cada release. Mantené tono consistente con el copy de arriba (minimalista, ligeramente ominoso).
- **A/B test** primero el screenshot principal y el título. La descripción larga rara vez mueve install rate; los assets visuales sí.
